Class: InputField

InputField(schema_name, data_statusopt)

Master class to represent input fields. Only the child classes are actually instantiated.

 draft Status of the schema version that the field is attached to, for form identification purposes.
Properties:
Name Type Description
id String Identifier of the field in relation to other fields, used in the DOM elements related to it. It matches `field_id` unless the field is new.
field_id String ID of the field as it will show in the "ID" input field of the editing form (empty if it has not been defined).
form_type String Internal type to distinguish different subclasses in the DOM IDs.
description String Description to show in the example of the options viewer. For now an empty string except in composite fields.
dummy_title String Title for the example in the options viewer. Always "Informative label".
mode String In first instance, "add"; when an existing field can be edited, "mod".
is_duplicate Boolean Whether the field has just been created via duplication of a different field.
schema_name String Name of the schema the field belongs to.
schema_status String Derived status of the schema as it is used in the ID of the form ('new', 'draft', 'copy' or 'object...').
required Boolean Whether the field should be required when implementing the metadata.
repeatable Boolean Whether the field can be repeated in the implementation form.
values Object Variable properties specific to different kinds of fields.
default String | Number Default value of the field, if the field is required.
form_field BasicForm Form to edit the contents of the field.
modal bootstrap.Modal Modal to which the editing form is attached.
button_title String User-facing of the type of form.
